# 函数组件

  • 通过属性来传递参数

  • 通过 props 来接收参数

function Hello(props) {
  return  (
    <div>props: {props.name}</div>
  )
}

ReactDOM.render(<Hello name='xiaosheng'></Hello>, document.getElementById('root'))

# 类组件

  • 通过属性来传递参数

  • 通过 this.props 来接收参数

import Hello from './Hello'

class Hello extends React.Component {
  render() {
    return (
      <div>{this.props.name}</div>
    )
  }
}

ReactDOM.render(<Hello name='xiaosheng'></Hello>, document.getElementById('root'))

# props 特点

  • 可以传递任意类型的参数, 如字符串, 数组, 数组, 函数, JSX 等
  • props 是只读属性, 不能修改属性的值
  • 在类组件中, 如果使用了构造函数, 则需要将 props 传递给 super() 函数, 否则的话将无法在构造函数中使用 this.props 来拿到传递过来的内容, 但是在其他函数中仍然可以正常使用 this.props

# children

  • children 属性表示组件标签的子节点, 当组件标签有子节点时, props 就会有该属性, 有点类似 vue 中的插槽

  • children 属性和普通的 props 一样, 值可以是任意类型 (文本, 组件, 函数等)

class Parent extends React.Component {
  render() {
    return (
      <div>
        {this.props.children} // 最后显示结果是 123
      </div>
    )
  }
}

ReactDOM.render(<Parent>123</Parent>, document.getElementById('root'))

    this.setState({
      childMsg:data
    })

  }

  render() {
    return (
      <div>
        <Child sendMsg={this.getChildMsg}></Child>

      </div>
    )
  }
}

class Child extends React.Component {

    props.sendMsg('child')
  }

  }

# 兄弟组件

  • 将共享状态提升到最近的一个公共父组件中, 由公共父组件来管理这个状态
  • 公共父组件的职责:
    • 提供共享状态
    • 提供操作共享状态的方法
    • 子组件通过 props 来接收状态或者调用方法
class Parent extends React.Component {

  state = {
    count: 0
  }

  changeValue = () => {
    this.setState({
      count: this.state.count + 1
    })
  }

  render() {
    return (
      <div>
        <Child1 count={this.state.count}></Child1>
        <Child2 changeValue={this.changeValue}></Child2>
      </div>
    )
  }
}

class Child1 extends React.Component {
  render() {
    return (
      <div>计数器:{this.props.count}</div>
    )
  }
}

class Child2 extends React.Component {
  render() {
    return (
      <button onClick={this.props.changeValue}>+1</button>
    )
  }
}

ReactDOM.render(<Parent></Parent>, document.getElementById('root'))

# 多级组件

使用 Context

  1. 调用 React.createContext() 来创建 Provider (提供数据) 和 Consumer (消费数据) 组件
  2. 使用 Provider 组件作为父节点
const {Provider, Consumer} = React.createContext()
class Parent extends React.Component {
  render() {
    return (
      // value 就是要传递的值
      <Provider value='parent'>
        <div>
          <Child1></Child1>
        </div>
      </Provider>
    )
  }
}

class Child1 extends React.Component {
  render() {
    return (
      <Child2></Child2>
    )
  }
}

class Child2 extends React.Component {
  render() {
    return (
      <Consumer>
        // data 就是传递过来的值, 大括号中必须是一个回调函数
        {data => <p>parent 传递过来的值: {data}</p>}
      </Consumer>
    )
  }
}

ReactDOM.render(<Parent></Parent>, document.getElementById('root'))

# 组件逻辑复用

# render props 模式

class Parent extends React.Component {
  getMouse = (mouse) => {
    return (
      <p>{mouse.x}</p>
    )
  }

  render() {
    return (
        <div>
          <Child1 render={this.getMouse}></Child1>
        </div>
    )
  }
}

class Child1 extends React.Component {
  state = {
    x: 0,
    y: 0
  }

  handleMove = (e) => {
    this.setState({
      x: e.clientX,
      y: e.clientY
    })
  }

  componentDidMount() {
    window.addEventListener('mousemove', this.handleMove)
  }
    
  componentWillUnmount() {
    window.removeEventListener('mousemove', this.handleMove)
  }

  render() {
    // 此处 render 也可以自定义名称, 和使用组件时传递过来的属性名保持一致
    return this.props.render(this.state)
  }
}

ReactDOM.render(<Parent></Parent>, document.getElementById('root'))

# 高阶组件

  • 使用步骤:

    1. 创建一个函数, 约定以 with 开头
    2. 指定该函数的参数, 参数以大写字母开头 (该参数作为要渲染的组件, 名字自定义即可)
    3. 在该函数内部创建一个类组件, 在类组件中提供复用的逻辑代码, 并将该类返回
    4. 在类组件中, 可以将参数当做属性进行传递
    5. 定义要显示的 UI 结构, 然后将其传递到上面创建的函数中, 然后就会接收到一个组件, 进而就可以使用该组件了

  • 注意事项:

    • 如果在多个地方使用高阶组件, 那么渲染出来的标签名称是一样的, 不方便调试, 所以需要设置 displayName 来区分不同的组件
// 相当于说是该函数定义了 js 逻辑
function withMouse(Component) {
  class Mouse extends React.Component {
    state = {
      x: 0,
      y: 0
    }

    handleMove = (e) => {
      this.setState({
        x: e.clientX,
        y: e.clientY
      })
    }

    componentDidMount() {
      window.addEventListener('mousemove', this.handleMove)
    }

    componentWillUnmount() {
      window.removeEventListener('mousemove', this.handleMove)
    }

    render() {
        return <Component {...this.state} {...this.props}></Component>
    }
  }
    
  Mouse.displayName = `WithMouse${getDisplayName(Component)}`


  return Mouse
}

function getDisplayName(Component) {
  return Component.displayName || Component.name || 'Component'
}

// 在这里定义了 HTML 结构, 因为是用来复用 js 逻辑的, 所以 html 结构就有可能不同
const position = (props) => {
  return (
    <p>鼠标的坐标x {props.x}</p>
  )
}

// 最终返回一个组件, 包含了 js 逻辑和 HTML 结构
const MousePosition = withMouse(position)

class App extends React.Component {
  render() {
    return (
      <MousePosition a={1}></MousePosition>
    )
  }
}

ReactDOM.render(<App></App>, document.getElementById('root'))

# 组件更新机制

  • 如果组件 A 更新之后, 则组件 A 的所有子组件, 孙子组件都会更新, 但是组件 A 的父组件, 兄弟组件则不会更新, 也就是说只会影响子组件

# 组件性能优化

# 减轻 state

  • state 中只存储和组件渲染相关的数据
  • 和渲染无关的数据不要放在 state 中
  • 需要在多个方法中使用的数据, 应该放在 this 中

# 减少不必要的重新渲染

  • 使用钩子函数 shouldComponentUpdate(nextProps, nextState), 通过返回值来决定该组件是否要重新渲染, true 表示需要, false 表示不需要
  • 该函数参数 nextProps,nextState 分别表示最新的 props 和最新的 state
  • 该钩子函数是在组件重新渲染前执行, 先执行该函数, 然后再执行 render 函数
  • 如果该钩子函数返回 false, 那么 render 函数也就不会执行
