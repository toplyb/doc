# 函数组件

  • webStorm 中可以使用 rsc 快速生成函数组件 (不带 props), 用 rsi 也可以生成函数组件, 是带 props
  • 使用函数创建的组件叫函数组件, 又叫做无状态组件
  • 只负责数据的展示
  • 函数名首字母必须大写, 如果是小写的, 则被识别为普通的 react 元素, 大写才会被识别成组件
  • 函数必须有返回值, 返回的是该组件的 HTML 结构, 如果不需要渲染 HTML, 则返回 null
  • 如果返回值为 null, 表示什么都不渲染
function Hello() {
  return (
    <div>第一个函数组件</div>
  )
}
// 或者使用箭头函数也可以
// const Hello = () => <div>第一个函数组件</div>

ReactDOM.render(<Hello></Hello>, document.getElementById('root'))
  • 或者也可以这样写
const App = () => {
  const clickHandler = (event) => {
    // event 是事件对象, 但他并不是原生的事件对象, 是经过 react 包装过的, 所以是在使用过程中, 不需要考虑兼容性问题
    // 取消默认行为
    event.preventDefault()
    
    // 关闭事件冒泡
    event.stopPropagation()
    console.log(event)
    console.log('aaa')
  }
  
  return (
    <div onClick={clickHandler}></div>
  )
}

# 类组件

  • 在 webStorm 中可以使用 rcc 快速生成一个类组件

  • 使用类创建的组件叫类组件, 又叫做有状态组件

  • 有自己的状态, 负责更新 UI, 让页面动起来

  • 类名首字母必须大写

  • 类组件必须继承 React.Component, 从而可以使用父类中提供的属性和方法

  • 类组件中必须有 render 方法, 该方法返回 HTML 结构

  • 在函数组件中, 使用 useRef 来获取到 dom 对象, 在 类组件中, 可以使用 React.createRef() 来获取到 dom

  • 类组件中自定的方法应该尽量使用箭头函数, 避免引起 this 指向问题

class Hello extends React.Component {
  // 这样的话, divRef.current 就可以拿到 dom 对象
  divRef = React.createRef()
    
  render() {
    return (
      <div ref={this.divRef}>这是第一个类组件</div>
    )
  }
}

// 也可以这样写
import { Component } from 'react'
class Hello extends Component {
  render() {
    return (
      <div>这是第一个类组件</div>
    )
  }
}

ReactDOM.render(<Hello></Hello>, document.getElementById('root'))

# 导入单独类组件

  • 当组件的代码写在一个单独的 js 文件中的时候, 可以使用以下方式
  • Hello.js
import React from "react"

class Hello extends React.Component {
  render() {
    return (
      <div>这是第一个类组件</div>
    )
  }
}

export default Hello
  • index.js
import Hello from "./Hello"

ReactDOM.render(<Hello></Hello>, document.getElementById('root'))

# 生命周期

  • 只有类组件才有生命周期

  • 生命周期分为三个阶段: 创建时 (挂载阶段), 更新时, 卸载时

  • 创建时常用的钩子函数有 (按照执行顺序排): constructor, render, componentDidMount

  • 更新时常用的钩子函数有 (按照执行顺序排): render, componentDidUpdate

  • 卸载时常用的钩子函数有 (按照执行顺序排): componentWillUnmount

# constructor

  • 组件创建时, 最先执行

  • 作用:

    1. 初始化 state
    2. 给事件绑定 this

# render

  • 每次渲染组件时都会触发, 该函数贯穿了组件的创建和更新两个阶段
  • 执行的时机:
    • 当组件接收到新属性的时候会触发该方法
    • 调用 setState 方法时
    • 调用 forceUpdate (强制更新) 方法时
  • 作用:
    1. 渲染 UI
    2. 不能在 render 中 调用 setState , 因为 setState 不仅能更新状态, 也能更新 UI, 如果调用 setState 的话, 会形成无限递归调用

# componentDidMount

  • 当组件挂载 (DOM 渲染完) 后执行
  • 作用:
    1. 发送网络请求
    2. 进行 DOM 操作

# componecntDidIpdate

  • 当组件更新后执行
  • 作用:
    1. 发送网络请求
    2. 进行 DOM 操作
    3. 在该函数中不管是直接调用还是间接调用 setState 的时候, 需要添加上 if 条件判断, 否则会形成无限递归调用, if 条件判断一般是拿当前的 props 和更新前的 props 来做比较, 该函数中的第一个参数 prevProps 就表示更新前的 props

# componentWillUnmount

  • 当组件卸载 (即从页面中消失) 的时候执行
  • 作用:
    1. 执行清理工作, 如清理定时器等

# 事件绑定

  • 语法: on + 事件名称={事件处理代码}

  • 命名采用小驼峰命名, 如: onClick, onMouseEnter

// 类组件
class Hello extends React.Component {
  divClick(e) {
    // e 是事件对象, react 中叫做合成事件, 兼容所有的浏览器
    console.log('我被点击了')
  }

  render() {
    return (
      <div onClick={this.divClick}>这是第一个类组件</div>
    )
  }
}

ReactDOM.render(<Hello></Hello>, document.getElementById('root'))
                
// 函数组件
function Hello {
  function divClick(e) {
    console.log('我被点击了')
  }

  return (
	<div onClick={divClick}>这是第一个函数组件</div>
  )
}

ReactDOM.render(<Hello></Hello>, document.getElementById('root'))
