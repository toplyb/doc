# 简介

  • hooks 的本质是一套能够使函数组件更强大, 更灵活的钩子
  • 由于函数组件没有自己的状态, 所以从 react16.8 开始, 出现了 hooks, 可以使函数组件有自己的状态
  • hooks 只能在函数组件中使用
  • hooks 解决的问题:
    1. 组件逻辑复用的问题
      • 在引入 hooks 之前, 可以使用 mixi 混入, HOC 高阶组件, render-props 等模式来解决逻辑复用, 但都有自身的问题, 如 mixin 的数据来源不清晰, 高阶组件的嵌套问题
    2. class 组件自身的问题
      • class 组件大而全, 太重了, 不够灵活

# useState基本使用

  • useState(0) 传递的参数表示 count 的初始值, 该函数可以执行多次
  • 不能在 if/for/其他函数 中调用 useState, 只能在函数根中出现
import {useState} from "react";

function App() {

  // [count, setCount] 是一种解构赋值的写法, useState() 返回的值是一个数组
  // [count, setCount] 的顺序不可变, 名字可以自定义
  // setCount 函数是用来修改 count 的值, 他是直接生成一个新的值来替换旧值, 因此在函数内部不能写 count++
  // count 和 setCount 是互相绑定的, setCount 只能用来修改 count
  const [count, setCount] = useState(0)
  
  const countAddHandler = () => {
      // 当用到之前的值, 建议传一个回调函数
      setCount(prevState => prevState + 1)
  }
  return (
    <div>
      <button onClick={() => setCount(count - 1)}>{count}</button>
      <button onClick={countAddHandler}>+1</button>
    </div>
  )
}

export default App;
  • 如果初始 state 需要通过计算才能获得, 则可以向 useState 中传递一个函数, 在函数中计算并返回初始 state, 此函数只在初始渲染的时候被调用
  • 如果值发生改变的话, 会导致组件重新渲染

# useEffect

  • useEffect 函数的作用是为 react 函数组件提供副作用处理的
  • 函数除了主作用外, 都是副作用, 主作用是指根据数据来渲染 UI, 除此之外, 都是副作用
  • 常见的副作用:
    1. 发送 ajax 请求
    2. 手动修改 dom
    3. localStorage 操作
  • 依赖项:
    • useEffect 默认是没有依赖项的, 没有依赖项的时候在组件初始化和重新渲染的时候都会执行
    • 可以接收第二个参数, 当第二个参数是 [] 作为空依赖项的时候, 副作用只会在组件第一次渲染的时候执行一次
    • 如下面的例子, 如果 useEffect 函数的第二个参数是 [count] 作为特定依赖项的话, 则只有 count 改变的时候会触发 useEffect, 而 name 改变的时候不会触发
  • 注意: 只要在 useEffect 函数中用到的数据状态, 就应该出现在依赖项中, 否则可能会有 bug
import {useState, useEffect} from "react";

function App() {
  const [count, setCount] = useState(0)
  const [name, setName] = useState('xiaoming')

  // 当通过修改状态更新组件时, 副作用也会执行
  useEffect(() => {
    document.title = count
  }, [count])

  return (
    <div>
      <button onClick={() => setCount(count + 1)}>{count}</button>
      <button onClick={() => setName('xiaosheng')}>{name}</button>
    </div>
  )
}

export default App;
  • 清除副作用: 当组件被销毁的时候, 如果有些副作用要被销毁, 如定时器, 则需要在 useEffect 中返回一个新的函数, 在返回的这个函数中清除掉副作用
import {useState, useEffect} from "react";

function Child() {
  useEffect(() => {
    const timer = setInterval(() => {
      console.log('aa');
    }, 1000)

    return () => {
      clearInterval(timer)
    }
  })
  return (
    <div>child</div>
  )
}

function App() {
  const [flag, setFlag] = useState(true)

  return (
    <div>
      {
        flag ? <Child></Child> : ''
      }
      <button onClick={() => setFlag(false)}>切换</button>
    </div>
  )
}

export default App;

# useRef

  • 只能用于函数组件或者自定义钩子函数

  • 钩子函数只能在函数组件中调用, 不能在函数组件中的自定义方法或者函数中使用

  • useRef 返回的就是一个普通的 js 对象, 所以我们可以创建一个普通的 js 对象, 也可以代替 useRef, 如 const dom = {current: null}, 但是我们创建的对象, 每次组件重新渲染的时候, 都会生成一个新对象, 但是使用 useRef 的话, 使用的都是同一个对象

  • 当需要一个对象不会因为组件的重新渲染而发生改变的时候, 就可以使用 useRef

  • 在 hooks 写法下通过 useRef 来获取到 dom

import React, {useEffect, useRef} from "react";

class Temp extends React.Component {

  getName = () => {
    return 'getname method'
  }

  render() {
    return (
      <div>
        temp
      </div>
    );
  }
}

function App() {
  const dom = useRef(null)
  const h1Dom = useRef(null)

  useEffect(() => {
    // 通过 current 可以获取到对应的 dom
    console.log(dom.current);
    console.log(h1Dom.current);
  }, [])

  return (
    <div>
      <Temp ref={dom}></Temp>
      <h1 ref={h1Dom}>test</h1>
    </div>
  )
}

export default App;

# useContext

import React, {createContext, useState, useContext} from "react";

const Context = createContext()

function Child1() {
  return (
    <div>
      <Child2></Child2>
    </div>
  )
}

function Child2() {
  const count = useContext(Context)

  return (
    <div>
      根组件传递过来的数据为 {count}
    </div>
  )
}

function App() {
  // 根组件中数据如果改变的话, 那么在其他组件中用到的该数据也会随之变化
  const [count] = useState(0)

  return (
    <div>
      <Context.Provider value={count}>
        <Child1></Child1>
      </Context.Provider>
    </div>
  )
}

export default App;
