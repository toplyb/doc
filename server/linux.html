# Linux 版本

# 内核版本

  • 主要由主版本号, 次版本号, 末版本号组成
  • 次版本号是奇数为开发版, 偶数为稳定版

# 常见目录

  • / 根目录
  • /home/用户名 普通用户的家目录
  • /root root用户的家目录
  • /etc 配置文件目录
  • /bin 命令目录
  • /sbin 管理命令目录
  • /usr 应用程序装在该目录
  • /usr/bin, /usr/sbin 系统预装的其他命令
  • boot 跟计算机启动相关的文件都放在该目录下
  • /dev 硬件, 键盘, 鼠标, 光驱之类的硬件放在该目录下

# 打包和压缩

# 打包

  • tar cf /tmp/etc.tar /etc 将 /etc 目录打包成 /tmp/etc.tar

  • 如果要压缩的话, 需要加参数 z, 如下:

    tar czf /tmp/etc.tar.gz /etc 或者 tar cjf /tmp/etc.tar.bz2 /etc

    • bz2 的压缩比例会更好, gzip 的压缩速度会更快

# 解包

  • tar xf /tmp/etc.tar -C /root 针对的是以 .tar 结尾的
  • tar zxf /tmp/etc.tar.gz -C /root 针对的是以 .gz 结尾的
  • tar jxf /tmp/etc.tar.bz2 -C /root 针对的是以 .bz2 结尾的

# vim 操作

# 光标操作

gg: 移动到首行的开头

G: 移动到最后一行的开头

shift + ^: 移动到当前行的开头

shift + $: 移动到当前行的开头

数字 G: 定位到某一行 (数字指的是某一行的行号)

ctrl + b: 向上翻一屏

ctrl + f: 向下翻一屏

ctrl + u: 向上翻半屏

ctrl + d: 向下翻半屏

# 可视化

  • 当输入 v 的时候进入可视化模式, 此时就可以进行选择文本内容, 相当于鼠标左键的选中状态, 然后使用上下左右箭头即可实现选中, 进而就可以进行复制粘贴等一系列操作

# 复制 / 粘贴 / 剪切

yy: 复制当前光标所在行

y: 当使用可视化模式选中内容的时候, y 表示复制

数字 yy: 表示从当前行开始, 向下复制数字行

p: 粘贴, 粘贴的内容是在当前光标的下一行

dd: 剪切或者删除当前行, 执行该操作后, 下面一行会向上移动

d: 当使用可视化模式选中内容的时候, d 表示剪切或者删除

数字 dd: 从当前行开始, 向下剪切或者删除数字行

D: 删除当前行并且会留空, 下面一行并不会向上移动

# 撤销 / 恢复

u: 撤销

ctrl + r: 恢复

# 查找

n: 向下翻页

N: 向上翻页

  • 当查找后, 查找的内容会高亮, 如果去掉高亮的话, 则可以再次搜索一次, 只不过这次搜索的内容可以随便打, 只要搜索不出来内容即可

# 替换

: 行号1, 行号2 s/内容1/内容2/g: 行号 1 和行号 2 表示要替换这两行之间的内容, s 表示替换, 固定的命令, 内容 1 表示文本中需要替换的内容, 内容 2 表示需要替换成的新内容, g 表示范围的都替换, 如果不加的话, 表示只会替换第一个

# 另存为

:w 路径/文件名

# 其他

set list: 显示特殊符号

set nonu: 取消显示行号

set nu: 显示行号

# 查看用户信息

  • 一个用户可以属于很多个组, 在创建用户的时候, 会随之创建一个同名的组, 该组就是该用户的基本组, 基本组只会有一个, 如果该用户还加入其它组的话, 那么其他组就是该用户的附加组

  • cat /etc/passwd

  • grep 用户名 /etc/passwd

  • head -1 /etc/passwd

    ![image-20210116220821245](/Users/xiaosheng/Library/Application Support/typora-user-images/image-20210116220821245.png)

    • 上图中内容的意思如下:

      用户名:x:uid:gid:描述:家目录:shell

      • x 是个占位符, 代表密码, 具体的内容不在这里, 有时候会是 *, 这时表示账号被锁定, 若是 !!, 则表示密码已过期
      • uid: 用户 id, 其中 若为 0, 则表示 root 用户, 1~999 表示系统用户, 普通用户是 1000+
      • gid: 用户组 id, 指的是基本组

    • 密码若是 $6$ 开头, 则表示是用 SHA-512 加密的; 若是 $1$, 则表示是 MD5 加密, 若是 $2$, 则表示 Blowfish 加密, 若是$5$, 则表示 SHA-256 加密

# 进程管理

# 查看进程

# ps

  • 静态查看进程
ps aux  # 查看所有进程
ps aux | head -2 # 查看第一个进程
ps a # 显示现行终端中的所有程序
ps u # 以用户为主的格式来显示程序
ps x # 不以终端机来区分
ps aux --sort -%cpu # 以CPU占比来降序排列
ps aux --sort %cpu # 以CPU占比来升序排列
ps -ef # 查看进程间的父子关系
ps axo user,pid,%cpu | head -2 # 自定义显示列

![image-20210223133756386](/Users/xiaosheng/Library/Application Support/typora-user-images/image-20210223133756386.png)

  • USER: 运行进程的用户
  • PID: 进程 id
  • CPU: CPU 占用率, 当达到百分之 1 以上的话, 就有点高了
  • MEM: 内存占用率
  • VSZ: 占用虚拟内存
  • RSS: 占用实际内存
  • TTY: 进程运行的终端名
  • STAT: 进程状态, Ss 表示睡眠状态
  • START: 进程开启的时间
  • TIME: 进程运行的时间
  • COMMAND: 命令的路径

# 定时任务

# crontab

  • crontab -e : 编辑定时任务

  • crontab -l : 查看定时任务列表

  • crontab -r : 删除定时任务

  • crontab 文件中的编写格式 :

    * * * * * 需要执行的命令 

命令解释:
第一个 * 表示分钟
第二个 * 表示小时
第三个 * 表示几号
第四个 * 表示月份
第五个 * 表示星期几

案例
*/2 * * * * echo $(date) >> /temp/date.log  # 表示每隔 2 分钟输出当前时间到 /temp/date.log 文件中

0 6-12/3 * 12 * /usr/bin/backup # 表示在 12 月内, 每天早上的6点到12点内, 每隔 3 个小时执行一次 /usr/bin/backup

0 6,12 * 12 * /usr/bin/backup # 表示在 12 月内, 在每天早上的6点和12点的时候, 执行一次 /usr/bin/backup

*/30 [3-5],[17-20] * * * /usr/bin/backup # 表示每天的3点到5点, 17点到20点的时间内, 每隔 30 分钟执行一次 /usr/bin/backup
